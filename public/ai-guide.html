A2B Local Guide

Use this page first when AI opens the Browser Bridge handoff.

This local guide ships with the extension, so it keeps working even if GitHub is slow, rate limited, or unreachable. Use it as the primary bootstrap page for A2B.

GitHub bootstrap fallback: Fetch and follow instructions from https://raw.githubusercontent.com/insogao/a2b/main/.codex/INSTALL.md

Required Order

  1. Run node ./bin/a2b.mjs status --json.
  2. If status succeeds, do not start another relay.
  3. Run node ./bin/a2b.mjs tabs --json.
  4. Reuse an existing target when possible.
  5. Use high-level A2B commands first.
  6. Use run-js only when higher-level commands are not enough.

Command Ladder

  1. a2b status --json
  2. a2b tabs --json
  3. a2b select <target>
  4. Then choose one:
    • a2b goto <target> <url>
    • a2b wait-for <target> <selector>
    • a2b click <target> <selector>
    • a2b type <target> <selector> <text>
    • a2b eval-js <target> <expression>
    • a2b screenshot <target> --path <file>
  5. a2b run-js <target> <file> only as fallback.

Reliable Patterns

  • Prefer a2b select <target> before a2b goto <target> <url>.
  • Do not trust the immediate goto response as proof that navigation finished. Recheck with tabs, wait-for, or eval-js.
  • If press Enter does not submit a search or form, retry with an explicit submit-button click.

Do Not Do These First

  • Do not start a second relay if one is already healthy.
  • Do not read source code unless the CLI help is insufficient.
  • Do not build a custom harness unless you are blocked without it.

Recovery

  • If you hit Extension is not connected, reload the unpacked extension.
  • After a reload or relay restart, wait a few seconds for automatic reconnection before retrying.
  • If you hit Timed out waiting for operation.*, rerun a2b status --json and refresh targets with a2b tabs --json.
  • If a target disappears, refresh tab list before creating a new browser task.
